Security ModelModelo de Seguridad

The agent can only do
what you allow.El agente solo puede hacer
lo que tú le permitas.

HandsAI isn't just a proxy — it's a permission boundary. Every layer is designed so that an agent operating in good or bad faith can't exceed what the human defined.HandsAI no es solo un proxy — es una frontera de permisos. Cada capa está diseñada para que un agente (de buena o mala fe) no pueda exceder lo definido por el humano.

  • 01

    No self-injection of toolsSin auto-inyección de herramientas

    An agent cannot register new APIs or modify existing ones. Only the human operator can, via the admin UI or the import API.Un agente no puede registrar o modificar APIs. Solo el operador humano puede configurarlas, mediante la UI o la API de importación.

  • 02

    Opaque credential managementGestión opaca de credenciales

    Credentials are AES-encrypted in PostgreSQL and decrypted only at dispatch time, in-flight. They never appear in tool responses or LLM context.Las credenciales se cifran (AES) en PostgreSQL y solo se descifran en vuelo al despachar. Nunca aparecen en las respuestas ni en el contexto del LLM.

  • 03

    PAT-protected bridgePuente protegido por PAT

    The Go bridge requires a Personal Access Token on every call to HandsAI. Unauthenticated requests are rejected with 401 before any tool executes.El puente de Go requiere un Token de Acceso Personal (PAT) en cada llamada. Las solicitudes no autenticadas se rechazan antes de ejecutarse.

  • 04

    Egress scrubbingLimpieza de salida (Egress)

    LLM-provided parameters are scanned for secrets before reaching any external API. If an agent accidentally — or maliciously — includes a token, it's redacted.Los parámetros del LLM se escanean en busca de secretos antes de alcanzar APIs externas. Si un agente incluye un token, este se redacta.

  • 05

    Injection-resistant responsesRespuestas herméticas a inyecciones

    All external content is tagged as untrusted and sanitized for prompt injection patterns before being returned to the LLM's context window.Todo contenido externo se etiqueta como no confiable y se sanean los patrones de prompt injection antes de devolverse al LLM.
